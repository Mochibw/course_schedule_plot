**Role (角色):**
你是一个专业的数据分析师和 Python 开发者。

**Task (任务):**
请读取我上传的课程表 PDF 文件（或图片内容），提取其中的课程信息，并将其整理为一个 Python 列表格式（`raw_data`）。

**Extraction Rules (提取规则 - 重要):**
请严格按照以下步骤处理数据，特别是针对“同一门课分段上课”的情况：

1.  **基本字段**：
    提取以下5个字段：`[课程名称, 教师, 星期(数字1-7), "开始节次-结束节次", 教室]`。

2.  **合并逻辑 (Merge Logic) - 必须执行**:
    检查提取的数据，如果发现 **两行或多行数据** 同时满足以下 **所有** 条件：
    - `课程名称` 相同
    - `星期` 相同
    - `节次` 相同
    - `教室` 相同
    - 但是 `起止周次` 不同（例如一个是1-8周，一个是9-16周）

    **请将它们合并为一条记录**。
    - **合并方式**：将“教师”字段修改为组合格式。
    - **格式示例**：如果原数据是 `孙慧` (1-8周) 和 `郭强辉` (9-16周)，请在合并后的教师字段填入：`"孙慧(1-8)/郭强辉(9-16)"`。

3.  **互斥逻辑 (Do Not Merge)**:
    如果课程名和时间相同，但 **教室不同**（通常是不同的“组”），请 **不要合并**，保持为两条独立记录。

4.  **数据清洗**:
    - **星期**：请转换为数字（周一=1, 周二=2...）。
    - **节次**：格式统一为 "X-Y"（例如 "7-9" 或 "3-4"）。
    - **周次**：如果是普通的连续周次（如1-16周）且没有分段老师的情况，教师名字后不需要加周次备注。仅在发生上述“合并”操作时，或有单双周特殊情况时，才在教师名后标注周次。

**Output Format (输出格式):**
请直接输出一段 Python 代码，包含名为 `raw_data` 的列表。格式如下：

```python
raw_data = [
    # [课程名, 教师(含周次), 星期, 节次, 教室]
    ["微分几何", "王险峰", 3, "7-8", "A411"],
    ["组合论", "孙慧(1-8)/郭强辉(9-16)", 3, "7-9", "A601"], # 合并示例
    ["高等代数", "李宁", 3, "7-8", "A203(组1)"], # 平行班不合并示例
    # ... 其他所有课程
]
```

请确保所有课程都被提取，没有遗漏。
